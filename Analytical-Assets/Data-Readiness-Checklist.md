# Data Readiness Checklist for AI-BAs

Use this checklist to evaluate if a department is ready for an AI implementation.

## 1. Data Availability
- [ ] Are data sources accessible via API?
- [ ] Is there a data catalog or dictionary available?
- [ ] Can the agent access real-time event streams?

## 2. Data Quality
- [ ] Is the data cleansed of duplicates?
- [ ] Is there a ground-truth dataset for validation?
- [ ] Are unstructured files (PDFs) machine-readable (OCR)?

## 3. Security & Governance
- [ ] Is PII (Personally Identifiable Information) masked or encrypted?
- [ ] Does the BA have clear data lineage documentation?
- [ ] Are there existing role-based access controls (RBAC)?

## 4. Organizational Readiness
- [ ] Is there a subject matter expert (SME) available for HITL review?
- [ ] Has the legal team approved the usage of this data for LLM grounding?
